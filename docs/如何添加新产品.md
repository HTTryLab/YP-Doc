本文档详细介绍如何向 YP 产品中心添加新的软件产品展示。

## 快速开始

### 使用自动创建脚本（推荐）

我们提供了交互式脚本来快速创建新产品：

```bash
node create-product.js
```

按照提示输入产品信息即可自动创建文档结构。

### 手动创建

如果您想手动创建，请参考 [完整步骤](#完整步骤)。

## 完整步骤

### 第一步：准备产品信息

在开始之前，准备好以下信息：

| 项目 | 说明 | 示例 |
|------|------|------|
| 产品名称 | 产品的完整名称 | YP.Storage |
| 产品标识 | 英文短标识，用于 URL 和目录 | storage |
| 产品图标 | Emoji 图标 | 📦 |
| 主题颜色 | 十六进制颜色代码 | #10b981 |
| 简短描述 | 一句话描述 | 企业级存储解决方案 |
| 详细描述 | 完整的产品描述 | 提供高可靠、高性能的云存储服务 |
| 核心特性 | 4-6 个关键特性 | 自动备份、版本控制、权限管理 |
| 官网地址 | 产品官网链接 | https://storage.example.com |

### 第二步：使用模板创建产品文档

#### 方法 1：使用脚本（推荐）

```bash
node create-product.js
```

#### 方法 2：手动复制模板

```bash
# 复制模板文件夹
cp -r product-template products/{product-slug}
```

然后替换模板中的占位符：

```bash
# 替换模板文件中的占位符
cd products/{product-slug}
# 手动编辑文件替换 {产品名称}, {product-slug} 等占位符
```

### 第三步：编辑产品文档

创建文档结构后，按照以下顺序编辑：

#### 1. 编辑产品简介 (`docs/intro.md`)

```markdown
# YP.Storage 企业级存储

YP.Storage 是一款提供高可靠、高性能的云存储解决方案。

## 核心特性
- 自动备份
- 版本控制
- 权限管理
- 跨平台同步
```

#### 2. 编辑快速开始 (`docs/getting-started/quickstart.md`)

```markdown
# 快速开始指南

## 前置要求
- Node.js 18+
- 磁盘空间 10GB+
```

#### 3. 编辑功能文档 (`docs/features/`)

为每个功能创建独立的文档文件。

#### 4. 配置侧边栏 (`sidebars.js`)

```javascript
const storageSidebar = {
  intro: [
    'intro',
    {
      type: 'category',
      label: '快速开始',
      items: [
        'getting-started/quickstart',
        'getting-started/installation'
      ],
    },
    {
      type: 'category',
      label: '核心功能',
      items: [
        'features/backup',
        'features/version-control',
        'features/permissions'
      ],
    },
  ],
};

module.exports = storageSidebar;
```

### 第四步：在 products.js 中添加产品配置

在 `products.js` 文件中添加新产品配置：

```javascript
const products = [
  // ... 现有产品
  
  {
    name: 'YP.Storage',
    slug: 'storage',
    description: '企业级存储解决方案',
    longDescription: '提供高可靠、高性能的云存储服务，支持自动备份、版本控制、权限管理等功能。',
    icon: '📦',
    color: '#10b981',
    docsUrl: '/products/storage/docs/intro',
    website: 'https://storage.example.com',
    features: [
      '自动备份',
      '版本控制',
      '权限管理',
      '跨平台同步'
    ]
  }
];

module.exports = products;
```

### 第五步：在 docusaurus.config.js 中添加插件配置

在 `plugins` 数组中添加新的文档插件：

```javascript
plugins: [
  // ... 现有产品插件
  
  [
    '@docusaurus/plugin-content-docs',
    {
      id: 'storage',
      path: 'products/storage/docs',
      routeBasePath: 'products/storage/docs',
      sidebarPath: require.resolve('./products/storage/sidebars.js'),
    },
  ],
]
```

### 第六步：更新导航栏

在 `docusaurus.config.js` 的 `themeConfig.navbar.items` 中添加：

```javascript
items: [
  {
    label: '产品首页',
    to: '/',
    position: 'left',
  },
  // ... 现有产品链接
  
  {
    label: 'YP.Storage',
    to: '/products/storage/docs/intro',
    position: 'left',
  },
  
  // ...
]
```

### 第七步：更新页脚

在 `docusaurus.config.js` 的 `themeConfig.footer.links` 中添加：

```javascript
links: [
  {
    title: '产品',
    items: [
      // ... 现有产品链接
      
      {
        label: 'YP.Storage',
        to: '/products/storage/docs/intro',
      },
    ],
  },
  
  // ...
]
```

### 第八步：清理缓存并重启服务器

```bash
npm run clear
npm start
```

## 产品配置详解

### products.js 配置项说明

| 配置项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `name` | string | ✅ | 产品完整名称 |
| `slug` | string | ✅ | 产品唯一标识 |
| `description` | string | ✅ | 简短描述 |
| `longDescription` | string | ✅ | 详细描述 |
| `icon` | string | ✅ | Emoji 图标 |
| `color` | string | ✅ | 主题色（十六进制） |
| `docsUrl` | string | ✅ | 文档链接 |
| `website` | string | ✅ | 官网链接 |
| `features` | array | ✅ | 特性列表 |

### 颜色选择建议

选择产品颜色时建议：

| 类型 | 推荐色 | 示例 |
|------|--------|------|
| 云计算/存储 | 蓝色系 | #2563eb |
| 数据/分析 | 绿色系 | #10b981 |
| AI/机器学习 | 黄色系 | #f59e0b |
| 安全/防护 | 红色系 | #ef4444 |
| 开发工具 | 紫色系 | #8b5cf6 |

### Icon 选择建议

可以在以下 Emoji 中选择：

- ☁️ 云/存储
- 📊 数据/分析
- 🤖 AI/机器学习
- 🔒 安全/防护
- 🚀 DevOps/工具

## 文档编写规范

### 目录结构标准

```
products/{product-slug}/
├── docs/
│   ├── intro.md                    # 产品简介（必填）
│   ├── getting-started/           # 快速开始
│   │   ├── quickstart.md           # 快速开始（必填）
│   │   ├── installation.md         # 详细安装
│   │   └── first-steps.md         # 入门教程
│   ├── features/                  # 功能文档
│   │   ├── feature1.md
│   │   └── feature2.md
│   ├── api-reference/              # API 文档
│   │   └── overview.md
│   └── release-notes.md           # 版本说明
└── sidebars.js                     # 侧边栏配置（必填）
```

### Markdown 文档标准

#### 1. 文件头示例

```markdown
---
id: intro
title: YP.Storage 简介
sidebar_label: 简介
---
```

#### 2. 标题层次

```markdown
# 一级标题（仅一个）
## 二级标题
### 三级标题
#### 四级标题
```

#### 3. 代码块

```bash
# 代码示例
command --option value
```

#### 4. 表格

| 列1 | 列2 | 列3 |
|------|------|------|
| 数据1 | 数据2 | 数据3 |
```

#### 5. 提示框

```markdown
:::tip 提示
这是提示内容
:::

:::warning 警告
这是警告内容
:::

:::danger 危险
这是危险内容
:::
```

## 高级配置

### 自定义产品首页

如果需要产品拥有独立的首页（不是文档首页），可以在 `src/pages/products/` 下创建：

```
src/pages/products/{product-slug}/
└── index.js
```

### 集成产品博客

为产品添加博客功能：

```javascript
[
  '@docusaurus/plugin-content-blog',
  {
    id: 'blog-{product-slug}',
    path: 'products/{product-slug}/blog',
    routeBasePath: 'products/{product-slug}/blog',
    blogTitle: '{产品名称}博客',
  },
]
```

### 添加产品版本控制

多版本文档可以使用 Docusaurus 的版本控制功能：

```javascript
{
  id: '{product-slug}',
  path: 'products/{product-slug}/docs',
  routeBasePath: 'products/{product-slug}/docs',
  includeCurrentVersion: true,
  versions: {
    current: {
      label: 'Next',
      path: 'next',
    },
  },
}
```

## 常见问题

### Q: 创建产品后不显示在首页？

**A:** 请检查以下项目：

1. ✅ `products.js` 中已正确配置产品
2. ✅ 重启了开发服务器
3. ✅ 清除了缓存 (`npm run clear`)

### Q: 文档路由 404 错误？

**A:** 请确认：

1. `docusaurus.config.js` 中 `routeBasePath` 配置正确
2. 侧边栏文件中的文档路径正确
3. 目录结构和文件名匹配

### Q: 如何修改产品卡片样式？

**A:** 修改 `src/pages/index.module.css` 中的 `.cardInner` 和相关样式。

### Q: 支持中英文双语吗？

**A:** 支持！请参考 Docusaurus 国际化文档配置 `i18n`。

### Q: 产品图片放在哪里？

**A:** 放在 `static/img/products/{product-slug}/` 目录下。

### Q: 如何删除一个产品？

**A:** 删除步骤：

1. 删除 `products/{product-slug}/` 目录
2. 从 `products.js` 中移除配置
3. 从 `docusaurus.config.js` 中移除插件和导航
4. 运行 `npm run clear`
5. 重启服务器
